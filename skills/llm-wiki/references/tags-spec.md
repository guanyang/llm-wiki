# Tag System

Tags are selected by combining three dimensions:

## Domain Tags (technical/business domain the page covers)

`AI`, `software-engineering`, `testing`, `CI-CD`, `infrastructure`, `observability`, `frontend`, `backend`, `DevOps`, `architecture`, `programming-languages`, `knowledge-management`, `databases`, `middleware`, `distributed-systems`, `cloud-native`, `security`, `information-science`

## Type Tags (type of thing the page describes)

`framework`, `tool`, `pattern`, `practice`, `standard`, `concept`, `person`, `project`, `methodology`, `comparative-analysis`, `synthesis`

## Maturity Tags (optional — only when clearly determinable)

`mature`, `emerging`, `experimental`, `deprecated`

## Rules

- Each page requires at least 1 domain tag + 1 type tag
- Maturity tags are optional — only annotate when there is clear evidence
- Prefer reusing existing tags; before adding a new one, check for synonyms that can be reused
- Tags use lowercase English with hyphens
